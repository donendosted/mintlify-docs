---
title: "Get Connector"
description: "Fetch metadata and status for a single connector."
openapi: "api-reference/v2/openapi.json GET /connectors/{id}"
---

<RequestExample>

```bash cURL
curl 'https://api.hydradb.com/connectors/{connector_id}' \
  -H "Authorization: Bearer $HYDRA_DB_API_KEY" \
  -H "API-Version: 2"
```

</RequestExample>

## Path parameters

| Name | Description |
| --- | --- |
| `id` | Connector UUID returned by `POST /connectors`. |

<ResponseExample>

```json 200
{
  "connector_id": "{connector_id}",
  "provider": "slack",
  "name": "acme-engineering",
  "tenant_id": "acme_corp",        // deprecated — use `database`
  "sub_tenant_id": "engineering",   // deprecated — use `collection`
  "provider_account_scope": "T12345ACME",
  "status": "active",
  "sync_status": "idle",
  "next_sync_at": "2026-06-01T13:00:00Z",
  "sync_interval_seconds": 3600
}
```

<Note>
The response includes `tenant_id` and `sub_tenant_id` for backward compatibility. The canonical names are `database` and `collection`. See [Migrating from `tenant_id` and `sub_tenant_id`](/essentials/v2/multi-tenant#7-migrating-from-the-legacy-tenant-and-sub-tenant-fields).
</Note>

</ResponseExample>
