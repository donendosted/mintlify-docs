---
title: "Update Source Metadata"
description: "Merge metadata onto an existing source without re-ingesting."
openapi: "api-reference/v2/openapi.json PATCH /context/{id}/metadata"
---

Merge or upsert `database_metadata` and `additional_metadata` onto a previously ingested source. Use this when you need to update metadata after ingestion — correcting a source's department tag, adding a priority flag, or enriching `additional_metadata` — without re-uploading the original content.

<Note>
This endpoint only updates metadata. The source's content and graph are not affected. To update content, re-ingest with the same `id` and `upsert: true`.
</Note>

<RequestExample>

```bash cURL
curl -X PATCH 'https://api.hydradb.com/context/{source_id}/metadata' \
  -H "Authorization: Bearer $HYDRA_DB_API_KEY" \
  -H "API-Version: 2" \
  -H "Content-Type: application/json" \
  -d '{
    "database": "acme_corp",
    "collection": "team_docs",
    "database_metadata": {
      "department": "legal",
      "priority": 7
    },
    "additional_metadata": {
      "reviewed_by": "alice",
      "doc_version": 3
    }
  }'
```

```python Python SDK
# The SDK does not expose this endpoint directly.
# Use the raw REST call:
import requests, json

response = requests.patch(
    f"https://api.hydradb.com/context/{source_id}/metadata",
    headers={
        "Authorization": f"Bearer {os.environ['HYDRA_DB_API_KEY']}",
        "API-Version": "2",
        "Content-Type": "application/json",
    },
    json={
        "database": "acme_corp",
        "collection": "team_docs",
        "database_metadata": {"department": "legal", "priority": 7},
        "additional_metadata": {"reviewed_by": "alice", "doc_version": 3},
    },
)
response.raise_for_status()
```

</RequestExample>

## Path parameters

| Name | Description |
| --- | --- |
| `id` | Source ID returned by `POST /context/ingest`. |

## Request body

| Name | Type | Description |
| --- | --- | --- |
| <code>database</code> | string | Target database. Required. |
| <code>collection</code> | string | Collection scope. Defaults to the default collection when omitted. Formerly `sub_tenant_id` (deprecated alias still accepted). |
| <code>database_metadata</code> | object | Schema-backed metadata fields to merge onto the source's `metadata`. Keys must match the database metadata schema. Formerly `tenant_metadata` (deprecated alias still accepted). |
| <code>additional_metadata</code> | object | Free-form key-value pairs to merge into the source's `additional_metadata`. |
| <code>tenant_id</code> | string | Deprecated — use `database`. |
| <code>sub_tenant_id</code> | string | Deprecated — use `collection`. |
| <code>tenant_metadata</code> | object | Deprecated — use `database_metadata`. |
| <code>document_metadata</code> | object | Deprecated — use `additional_metadata`. |

<Info>
**Merge semantics.** Fields are merged, not replaced. Sending `{"database_metadata": {"priority": 7}}` adds or updates the `priority` key while leaving other metadata keys intact. To remove a key, set it to `null`.
</Info>

<ResponseExample>

```json 200
{
  "success": true,
  "data": {
    "chunk_rows_matched": 1,
    "chunk_rows_modified": 1,
    "database_metadata_keys": ["department", "priority"],
    "additional_metadata_keys": ["reviewed_by", "doc_version"]
  },
  "error": null,
  "meta": {
    "request_id": "9d13aef4-02f4-4e73-8c62-4c2601d04f9d",
    "latency_ms": 8.2
  }
}
```

</ResponseExample>

| Field | Description |
| --- | --- |
| `chunk_rows_matched` | Number of chunk rows matched by the source ID. |
| `chunk_rows_modified` | Number of chunk rows actually modified. |
| `database_metadata_keys` | Metadata keys that were included in the update. |
| `additional_metadata_keys` | Additional metadata keys that were included in the update. |
