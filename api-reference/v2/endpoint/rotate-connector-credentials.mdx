title: "Rotate Connector Credentials"
description: "Rotate OAuth refresh token for a connector (internal endpoint)."
openapi: "api-reference/v2/openapi.json PATCH /connectors/{id}/credentials"
---

<Warning>
This is an **internal service endpoint** used by HydraDB's OAuth token refresh loop. It requires the `X-Cortex-Secret` header — a private internal credential, not your API key. Public API consumers should not call this endpoint directly.
</Warning>

HydraDB's sync engine may need to persist a rotated OAuth `refresh_token` for connectors that use OAuth-based authentication (Slack, GitHub, Notion, Gmail, Linear). Some providers invalidate the previous refresh token on each exchange, so the new token must be written back or the next sync fails with `invalid_grant`.

The endpoint merges the new `refresh_token` onto the connector's stored credential bundle and re-encrypts it under the connector's identity context.

<RequestExample>

```bash cURL
curl -X PATCH 'https://api.hydradb.com/connectors/{connector_id}/credentials' \
  -H "X-Cortex-Secret: <internal_secret>" \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "new_oauth_refresh_token_value"
  }'
```

</RequestExample>

## Path parameters

| Name | Description |
| --- | --- |
| `id` | Connector ID returned by `POST /connectors`. |

## Authentication

This endpoint does **not** use the standard `Authorization: Bearer` header. It requires the internal `X-Cortex-Secret` header, which is a private infrastructure credential. See your deployment's internal documentation for the value.

## Request body

| Name | Type | Description |
| --- | --- | --- |
| `refresh_token` | string | The new OAuth refresh token to persist. |

## Behavior notes

- Only `refresh_token` is merged onto the current stored credential bundle.
- The bundle is re-encrypted under the connector's identity context.
- Credentials are **never returned** by any API — this is a write-only operation.
- If the provider does not rotate refresh tokens, this endpoint is not needed.

```json 200
{}
```

</ResponseExample>
