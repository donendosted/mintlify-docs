List a provider's resources directly from supplied credentials, **before** creating a connector. Use this to validate credentials, see what's available, and verify scope before committing to a connector configuration.

<RequestExample>

```bash cURL
curl -X POST 'https://api.hydradb.com/connector-discovery' \
  -H "Authorization: Bearer $HYDRA_DB_API_KEY" \
  -H "API-Version: 2" \
  -H "Content-Type: application/json" \
  -d '{
    "provider": "slack",
    "credentials": {
      "api_token": "xoxb-..."
    },
    "auth_type": "api_token"
  }'
```

</RequestExample>

## Request body

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `provider` | string | yes | External provider to preview (e.g. `slack`, `github`, `linear`, `notion`, `gmail`). |
| `credentials` | object | yes | Provider-specific credentials, typically `{"api_token": "..."}` or `{"access_token": "..."}`. |
| `auth_type` | string | no | Authentication method (e.g. `api_token`, `oauth`). |
